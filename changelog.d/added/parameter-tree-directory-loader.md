Add `ParamTree`, a tree-shaped parameter loader that mirrors the Python `policyengine-uk` directory-tree parameter system (issue #50), landing additively alongside the existing hand-coded `Parameters` struct and flat year-files. `ParamTree::from_dir` walks a directory tree of YAML parameter files and parses each leaf into a typed node supporting the three Python node types — `values:` (date-keyed scalars), `brackets:`, and `scales:` (a synonym for brackets). Parameters resolve by dot-separated path (`get_at("gov.hmrc.income_tax.rates.basic_rate", 2025)`) with Python step-function date semantics (latest value on or before the requested date, with `null` entries clearing a window). Values requested beyond the last declared year are auto-uprated forward by an index declared in `metadata.uprating` (e.g. `gov.indices.cpi`), scaling the last-known value by the ratio of the index at the requested date to the index at the value's last-known date, and holding flat when no index is available. A small fixture parameter tree under `tests/fixtures/parameters/gov/` exercises value-by-date resolution, bracket/scale parsing, path lookup, and forward uprating projection. Migrating `compute_*` consumers off the hand-coded struct onto path-keyed tree lookups is the remaining follow-up for #50.
