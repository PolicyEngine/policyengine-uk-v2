Add `ParameterTree`, a tree-walking read path for parameter YAMLs that exposes any year's YAML as a generic `serde_yaml::Value` indexable by dot-separated path. Lets contributors add a parameter to YAML and read it out via `tree.lookup_f64("income_tax.personal_allowance")` without first mirroring it as a Rust struct field. Coexists with the canonical `Parameters::for_year`; existing behaviour unchanged. Sequence indices are written `[N]`, e.g. `income_tax.uk_brackets[1].rate`. First slice toward #50; future slices may rebuild `Parameters::for_year` on top of this loader, eventually walking a Python-style `parameters/gov/...` tree.
