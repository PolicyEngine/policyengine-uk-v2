Add an in-process PyO3 extension (`_native.so`, built with `cargo build --release --features python --lib`) so the Python wrapper no longer shells out to the binary for population runs. A native `Simulation` loads the dataset once and caches the baseline results and the baseline-side labour supply retention rates, so each `run(policy)` only pays the reform-dependent work; `Simulation.run()` in the Python wrapper uses it automatically for file-based data (falling back to the subprocess binary for stdin-data modes and structural reforms). The analysis pipeline moved from `main.rs` into a library (`src/run.rs`) shared by the binary and the extension, with output verified byte-identical. On the FRS population with labour supply responses, repeat reform scoring drops to ~0.3s per reform and a baseline run to ~0.15s including data load; the two-run UC reform example script drops from 3.8s to 1.6s end to end, most of which is now Python import time. pandas imports in the wrapper are deferred to first use to keep `import policyengine_uk_compiled` fast.
