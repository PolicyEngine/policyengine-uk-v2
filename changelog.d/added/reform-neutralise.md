Add `neutralise:` support to reform YAMLs. A top-level `neutralise:` sequence in a reform file zeros the named benefit outputs in the reform simulation, mirroring `policyengine_uk/reforms/policyengine/disable_simulated_benefits.py`. Twelve benefits are supported in this slice — `universal_credit`, `child_benefit`, `pension_credit`, `housing_benefit`, `child_tax_credit`, `working_tax_credit`, `income_support`, `esa_income_related`, `jsa_income_based`, `carers_allowance`, `scottish_child_payment`, `passthrough_benefits`. Aggregate fields (`total_benefits`, `net_income`, `equivalised_net_income*`, `extended_net_income`) are recomputed by delta so the result set stays consistent. Composes with parameter overrides in the same file. State pension and `benefit_cap_reduction` need cross-variable feedback and remain follow-up slices of #41 (the trait restructure + replace-formula support are the next two slices). Includes `reforms/disable_means_tested.yaml` as a worked example.
