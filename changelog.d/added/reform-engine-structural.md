Add a structural reform abstraction (`src/reforms/structural.rs`) that goes beyond the YAML parameter overlay, addressing #41. The new `StructuralReform` enum can neutralise a named benefit variable, override its computed output with a custom closure (`fn(old_value, &BenUnit) -> new_value`), and compose any number of these in sequence (`.then(...)`), in addition to wrapping the existing parameter-overlay `Reform`. Structural reforms are applied as a post-compute transform on the finalised reform-side `SimulationResults` — the same hook the existing neutralisation uses — reconciling benunit `total_benefits` and household income aggregates (`net_income`, `net_income_ahc`, `extended_net_income`, `equivalised_*`) by delta. Ships a small example registry (`disable_simulated_benefits`, `neutralise_benefit`, `halve_child_benefit`, `by_name`) analogous to the Python `policyengine_uk/reforms/` examples. The existing parameter-override and YAML `neutralise:` paths are unchanged. Full mid-pipeline `compute_*` formula replacement remains a follow-up.
