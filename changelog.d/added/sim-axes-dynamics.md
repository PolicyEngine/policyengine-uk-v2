Add **axes** (parameter / input sweeps) and a **dynamics** behavioural-response hook to the simulation engine, building on the existing `Simulation::branch` primitive. `Axis::on_people` / `Axis::on_people_where` / `Axis::on_parameters` describe a swept variable as `(name, min, max, count, setter)`; `Simulation::run_axis` linearly spaces the range, applies the setter per step, runs the engine in parallel via rayon, and returns one `AxisStep { index, value, results }` per point — the per-step results that back marginal-tax-rate, cliff-edge, and budget-constraint curves. The new `Dynamics` trait formalises the between-baseline-and-reform behavioural pass: `apply_dynamics` threads a sequence of passes over the input frame, with `LabourSupplyDynamics` (wrapping the existing OBR Slutsky-decomposition `apply_labour_supply_responses`, so the labour-supply formulas now have a formal hook — no logic duplicated) and `TakeUpDynamics` (a deterministic benefit take-up campaign that flips `would_claim_*` flags for benefit units whose `migration_seed` is below a configurable `target_rate`). Additive — existing engine behaviour and `main.rs` are unchanged. Addresses #49.
