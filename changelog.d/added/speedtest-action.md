Add a `Speedtest` GitHub Action that reports how the time to run the engine over 100k households changes in a PR (#69). A new self-contained `benches/speedtest.rs` (a `harness = false` binary, no FRS data required — synthetic single-adult frame, scale via `SPEEDTEST_HH`/`SPEEDTEST_RUNS`) times `Simulation::run()` and prints a parseable `SPEEDTEST_JSON=` line. The workflow runs it on the PR head and the base commit, then posts/updates a sticky PR comment with the median run time, throughput, and the delta vs base (gracefully reporting absolute timing only when the bench is absent from the base commit).
