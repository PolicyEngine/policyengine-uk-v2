Add the first slice of the axiom rules-engine backend (`src/axiom/`): policyengine-uk-rust can now evaluate statute-derived RuleSpec rules (composed from rulespec-uk) instead of hand-coded variable logic. `Policy` wraps a compiled artifact; reforms and future-dated uprating assumptions are parameter overrides (`Policy::with_parameter`) that patch the versioned parameter table and recompile in memory. `Dataset` maps per-person input columns onto one tax year, and `calculate` runs the engine's vectorised fast path, returning output columns in person order. Coverage is the s.10 rate-band surface only for now — the full s.23 liability composition is the next slice.
