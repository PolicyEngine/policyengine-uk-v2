Route `Simulation.run_microdata` through the in-process native engine instead of spawning a subprocess and round-tripping the microdata through CSV. The native path returns columnar buffers (full-precision float64, real bool, int64 ids) wrapped zero-copy via `np.frombuffer`, eliminating the dominant CSV encode/decode cost on large datasets such as EFRS. A single typed-column source in `clean.rs` feeds both the native path and the byte-identical CSV writers. The legacy `clean_frs_base`/`clean_frs`/`frs_raw` constructor kwargs are removed in favour of `data_dir`/`dataset`.
