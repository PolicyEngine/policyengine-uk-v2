State pension now respects the recorded amount for the new-SP cohort (those reaching SP age after April 2016), mirroring the existing basic-SP behaviour. Previously `person_state_pension` ignored the FRS-recorded `state_pension` for the new-SP branch and always returned the full parameter rate × 52, overstating state pension on households whose recorded amounts differ from the full new-SP rate. It now returns the recorded amount scaled by the reform ratio (`new_state_pension_weekly / baseline_new_sp_weekly`) when an amount is recorded, falling back to the full rate × 52 only when nothing is recorded. The baseline new-SP rate is threaded through `Simulation::new` / `new_with_baseline_sp` parallel to the existing baseline old-SP rate.
